# 常见问题解答 (FAQ)

## 在 iOS 视频播放器上不起作用

**问题**：在 iOS 全屏播放器上不起作用。

**回答**：[关于添加新 SSH 密钥的 GitHub 指南](https://docs.github.com/en/github/authenticating-to-github/connecting-to-github-with-ssh)。（注：原链接似乎与视频播放器无关，但保留原样。）

## 如何在模拟器上测试

**问题**：如何在模拟器上测试？

**回答**：请访问[此处](../testing/testing.md)获取更多信息。

## 3. 问题：测试失败

**问题**：即使我没有做任何更改，我的测试也失败了。

**回答**：这可能有几个原因：
1. 确保您已从主分支拉取了最新更改。
2. 通过运行 `npm install` 或 `yarn install` 检查是否有任何新的依赖项或需要更新。
3. 确保您的本地环境与存储库文档中描述的设置相匹配。

## 4. 问题：构建失败

**问题**：构建过程失败并报错。

**回答**：构建失败的常见原因包括：
1. 环境变量缺失：确保所有必需的环境变量都已正确设置。
2. 依赖问题：运行 `npm install` 或 `yarn install` 以确保所有依赖项都是最新的。
3. 配置问题：双击检查构建配置文件（如 `webpack.config.js`）或任何其他与构建相关的设置。

## 5. 问题：代码规范 (Linting) 错误

**问题**：如何修复代码中的规范错误？

**回答**：要修复代码规范错误，请运行存储库中提供的 linter 工具。例如，如果存储库使用 ESLint，您可以运行：

```sh
npm run lint --fix
```

或者

```sh
yarn lint --fix
```

这将自动修复许多常见的代码规范错误。如果仍然存在任何错误，请参考错误消息以获取有关如何解决它们的具体指导。

## 6. 问题：贡献指南

**问题**：我该如何为此存储库做出贡献？

**回答**：我们欢迎各种贡献！请参阅存储库根目录下的 CONTRIBUTING.md 文件，了解有关如何贡献的详细指南。它包括有关设置开发环境、编码标准以及提交拉取请求的过程的信息。

## 7. 问题：联系维护者

**问题**：我该如何联系此存储库的维护者？

**回答**：如果您需要联系维护者，可以在 GitHub 上发起一个 issue 并标记维护者。或者，检查存储库是否有专门的通信渠道，如 Slack、Discord 或邮件列表。

## 8. 问题：许可证信息

**问题**：此存储库的许可证是什么？

**回答**：此存储库根据 MIT 许可证进行授权。您可以在存储库根目录的 LICENSE 文件中找到更多详细信息。
